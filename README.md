# **springCloud新手练习demo**



 ##模块说明：
 ###eureka-server     作为注册中心   这个跟配置中心产生的原因也是一样 集中好管理
 ###eruka-client      服务提供者 将自己的服务注册到注册中心  
 ###eruka-client2     服务提供者 这里为更好为ribbon做负载均衡复制了一个一模一样的client项目 除了端口不一样其他地方都是一样的
 ###service-ribbon    客户端负载均衡组件
 ###service-zuul      路由代理服务  
 ###service-feign     服务间相互调用的这么一个组件。类似于RPC 。跨系统调用而已。
 ###service-register  配置中心 这个目前就没写了。项目产生总有个原因或者背景，谁没事闲得蛋疼没事搞这么个项目，这是因为项目越大
    运维难度越大，集群上的配置文件成百上千，急需要这个一个集中管理的地方，所以这个东西应运而生，可能我们开发感受不到。但是去运维的话，还是很难受的，原理就是项目每次启动的时候通过http的接口嘛，集中去拉取配置文件到项目本地， 就是这个原理。当然还有修改了东西，怎么更新。它提供了一个http的接口敲一下就更新了。

 ##启动说明：
 ###1.启动eureka-server  启动所有模块都需要到注册中心上拉取服务地址。所以启动是先启动注册中心的。
 ###2.启动eureka-client  理由很简单 你服务不启动。谁来支撑这个服务，启动多个 你就把service-client2也启动起来
 ###3.启动service-ribbon 启动这个负载均衡来访问提供服务者接口。访问这个来分发做一个负载均衡的请求 
 ###4.启动service-feign  这个feign有集成了ribbon 所以他是个负载均衡的。这个ribbon其实在项目里 基本上不单起应用了
